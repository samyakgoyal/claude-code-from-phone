01 ‚Äî What

What Is This?

This setup lets you run Claude Code (or any terminal session) on your Mac ‚Äî from your iPhone. From anywhere. Home, office, gym, train, bathroom stall during a standup you forgot to mute.

Your Mac stays plugged in and awake. Your phone becomes a window into its terminal. One command, you're in. Close the app, walk away, come back hours later ‚Äî session is exactly where you left it.

Tailscale
Private encrypted network between your devices. No open ports needed.
cost: ₹0
Termius
SSH client for iPhone. Terminal on your phone, but prettier.
cost: ₹0
tmux
Keeps sessions alive when you disconnect. Pause button for terminals.
cost: ₹0
SSH
The actual connection protocol. Built into macOS since forever.
cost: ₹0

Total cost: ₹0 ¬∑ Setup time: ~15 min ¬∑ Productivity lost to "let me just check one thing": immeasurable

How It Works

Tailscale gives both devices a fixed private IP (like 100.89.36.88) that works over any network. Devices talk directly peer-to-peer via WireGuard encryption.

SSH establishes an encrypted terminal session over that tunnel. Your Mac's port 22 is only reachable through Tailscale ‚Äî invisible to the public internet.

tmux is a session living in your Mac's RAM. SSH just "attaches" to it ‚Äî like plugging a monitor into a running computer. Disconnect? tmux keeps going. Reconnect? Right back where you were.

The Journey of a Single Keystroke

You tap 'l' on iPhone keyboard
Termius encrypts it SSH ¬∑ AES-256
iOS sends via Tailscale WireGuard ¬∑ ChaCha20
Packet travels: Phone → Internet → Router → Mac
Mac's Tailscale decrypts the WireGuard layer
Mac's SSH daemon decrypts the SSH layer
Plaintext 'l' arrives at tmux → Claude Code
Response travels back the same way, encrypted twice

At no point does your keystroke travel unencrypted. Not on the gym WiFi, not on cellular, not anywhere.

03 ‚Äî Setup

Setting It Up (15 min, one-time)

  1. Tailscale ‚Äî the network magic

    On Mac:

    Terminal
    brew install tailscale sudo brew services start tailscale sudo tailscale login # Browser opens → sign in with Google/Apple/GitHub # Grab your IP: sudo tailscale status # Note the 100.x.x.x address
    ⚠️
    On corporate VPN? Disconnect it first. VPNs and Tailscale get along like two cats in a studio apartment.

    On iPhone: App Store → download Tailscale → sign in with the same account. Done.

  2. Enable SSH on Mac

    System Settings → General → Sharing → Remote Login → ON

    Verify
    ssh localhost # Password prompt → logs in → you're good
  3. Termius on iPhone

    App Store → download Termius → add a new host:

    • Hostname: your Tailscale IP (100.x.x.x)

    • Username: whatever whoami outputs on your Mac (use the output, not the word "whoami" ‚Äî yes, this has happened)

    • Password: your Mac login password

    Tap connect → you're inside your Mac's terminal from your phone.

  4. tmux ‚Äî persistent sessions
    Install + Magic Alias
    brew install tmux # Set up the shortcut (run once): echo "alias sg='tmux attach -t claude || tmux new -s claude'" >> ~/.zshrc && source ~/.zshrc

    Now sg = instant Claude Code session. Attaches if one exists, creates if it doesn't.

  5. Keep Mac awake

    System Settings → Battery → Options → "Prevent automatic sleeping on power adapter" → ON

    Or via terminal
    sudo pmset -c sleep 0 disksleep 0 displaysleep 5

    Mac sleeps = sessions die = sadness. Keep it plugged in.

The "What Would It Take" Test

For someone to break in, they'd need all three simultaneously:

1. Compromise your Tailscale account (your Google/Apple login + 2FA)

2. Know your Tailscale IP (only visible to your devices)

3. Crack your SSH credentials (Mac password or private key)

If someone can pull all that off, they don't need your terminal ‚Äî they already own your life.

05 ‚Äî Troubleshooting

When Things Go Wrong

Because they will. Not "if" ‚Äî "when." Here's the greatest hits of "why isn't this working" and how to fix them.

Tailscale needs to be running on both devices. Open the Tailscale app on your iPhone and make sure it's connected. On your Mac, check the menu bar icon or run sudo tailscale status. If both show "online" but you still can't connect, try tailscale ping <your-mac-ip> from your phone's Termius to see if packets are getting through. If the ping fails, both devices might be on networks that block WireGuard ‚Äî Tailscale should fallback to relay, but give it a few seconds.
99% chance your Mac fell asleep. Check your pmset settings ‚Äî you want sleep 0 when on power adapter. Also watch out for clamshell mode: if your Mac's lid is closed, it will sleep unless it has an external display connected (or you're running caffeinate -s in the background). Pro tip: caffeinate -s & in your tmux session keeps the Mac awake as long as the session exists.
Your Mac restarted. tmux sessions live in RAM ‚Äî they don't survive reboots. macOS loves a good unsolicited restart (looking at you, automatic updates). Just run sg again to start a fresh session. If this keeps happening, go to System Settings → General → Software Update → Automatic Updates and turn off "Install macOS updates." You'll update when you decide, not when Apple does.
You're in a new shell that hasn't loaded your aliases. Run source ~/.zshrc and try again. If that doesn't work, check that the alias actually exists: grep "alias sg" ~/.zshrc. If it's not there, the setup step didn't stick ‚Äî run the echo command from Step 4 again. If you use bash instead of zsh, you need ~/.bashrc instead of ~/.zshrc.
Your traffic is probably going through Tailscale's DERP relay servers instead of a direct peer-to-peer connection. Run tailscale status on either device and look for the word "relay" next to the other device. Direct connections are way faster. This usually happens when both devices are behind strict NATs. Fix: try connecting to a different WiFi, or switch between WiFi and cellular on your phone. Tailscale will retry the direct connection. If you're stuck on relay permanently, consider Mosh instead of SSH (see Level Up section).
Some carriers block WireGuard's UDP traffic (port 41641). Tailscale should automatically fall back to DERP relay servers over HTTPS, but this can take a few seconds. If it never connects, force-close and reopen the Tailscale app on your phone. In rare cases, certain carriers in certain countries are just hostile to VPN-like traffic. Not much you can do except switch carriers or use a WiFi hotspot.
On your Mac (the server) ‚Äî yes. Tailscale and commercial VPNs (NordVPN, ExpressVPN, etc.) coexist fine since they use separate network interfaces. Corporate VPNs (Cisco AnyConnect, GlobalProtect, Zscaler) can be trickier ‚Äî they sometimes take over all routing and block other tunnels.

On your iPhone ‚Äî no. iOS only allows one VPN tunnel at a time. Turning on a commercial VPN will disconnect Tailscale, and vice versa. You have to pick one.

Workaround: Run the commercial VPN on your Mac instead. Your phone connects to your Mac via Tailscale, and your Mac's traffic goes through the VPN. Best of both worlds ‚Äî encrypted tunnel to your Mac and VPN protection on the Mac's outbound traffic.
tmux doesn't enable mouse/scroll support by default. Create or edit ~/.tmux.conf on your Mac and add set -g mouse on. Then reload with tmux source ~/.tmux.conf (or just detach and reattach). Scrolling, clicking, and pane resizing will all work after that. While you're at it, add set -g history-limit 10000 so you have a decent scrollback buffer.

07 ‚Äî Level Up

Power User Moves

Got the basics working? Good. Now let's make it actually comfortable. These are the tweaks that separate "I can use it" from "I prefer it."

  • SSH key auth instead of password security
    Typing your Mac password on a phone keyboard every time is a special kind of torture. Generate an SSH key pair on your phone (ssh-keygen -t ed25519 in Termius or Termux), copy the public key to your Mac's ~/.ssh/authorized_keys, and never type a password again. Bonus: it's more secure than passwords because you can't brute-force a 256-bit key. Disable password auth entirely in /etc/ssh/sshd_config if you're feeling bold.
  • Better .tmux.conf comfort
    Default tmux is functional but ugly. Add set -g mouse on for scroll support (essential on phone). Set setw -g mode-keys vi for vim-style navigation. Change the status bar to show useful info. Remap the prefix from Ctrl+B to Ctrl+A (easier to reach). There are great minimal configs on GitHub ‚Äî search "tmux sensible" for a solid starting point.
  • Mosh instead of SSH reliability
    If you're on flaky connections (cellular, train WiFi, that one coffee shop), Mosh is SSH's more resilient cousin. It handles network switches, high latency, and temporary disconnects gracefully. Your session doesn't die when you switch from WiFi to cellular. Install on Mac with brew install mosh, use Blink Shell on iPhone (it has built-in Mosh support). Downside: needs a UDP port open, which Tailscale handles fine.
  • Multiple named tmux sessions workflow
    Don't limit yourself to one session. Run tmux new -s work for work stuff, tmux new -s personal for side projects, tmux new -s claude for AI coding. Switch between them with tmux attach -t work. List all sessions with tmux ls. You can even create aliases for each: alias sw='tmux attach -t work || tmux new -s work'. It's like having multiple desktops, but in your pocket.
